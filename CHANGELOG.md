# Changelog

## v2.9.1 (2026-06-26)

**Compact re-anchor: actionable truncation marker + a corrected understanding of what survives.** Auditing a real 69 KB append-drifted `TASK.md` raised a fear that compaction survival fails on a bloated ledger. Investigation corrected an earlier overstatement: `buildDigest` surfaces `GOAL`/`NOW`/`NEXT` (parsed, first-wins) and blocked items as **separate fenced lines that always survive** ‚Äî only the `--- Full TASK.md ---` *convenience* embed is capped at `TASK_BODY_MAX_CHARS` (4000) and truncated from the end. So the live `NOW` one-liner is **not** lost on bloat, and the on-disk file is always intact.

- The truncation marker is now **actionable**: instead of `[‚Ä¶truncated to budget]` it reads "*open `.ai/memory/TASK.md` on disk for the full, current state*", so a re-anchoring agent reads the intact file rather than trusting a partial embed.
- The real fix for a bloated `TASK.md` is write hygiene (keep it lean), not a bigger cap ‚Äî recorded as a decision in the project's own ledger (dogfooding the new `code:` back-link + `aliases:` search terms). Non-blocking backlog: optionally hoist the checklist tail ahead of free-form notes before truncating.
- `selftest` +2 (marker points to disk; live `NOW` survives a bloated body) ‚Üí 76/76.

## v2.9.0 (2026-06-26)

**Relevance-decay machine fields + lesson lifecycle, plus a resident decision index that ends silent fragmentation.** Motivated by auditing a real non-author project (a 72 KB `DECISIONS.md` / 69 KB append-drifted `TASK.md`): the decay model was sound but had no explicit fields to act on, and the digest surfaced only the top-5 decision titles ‚Äî so an agent silently never knew the other 25 existed. The relevance-decay fixes were hardened by an adversarial Codex review (one real supersede bug + four guards).

1. **Resident index ‚Äî decisions AND lessons (anti-fragmentation).** The SessionStart/PreCompact digest now lists current decision *titles* **and active lesson titles** newest-first, each with a header that **always carries the total count** and the path to the rest (`decisions` for all titles, `search` for bodies) ‚Äî so the agent sees *which* decisions and pitfalls exist and can't silently miss one and re-litigate/repeat it, while bodies stay pull-on-demand. The map is resident; the territory loads on demand. Title caps `DIGEST_DECISIONS_TOPN` (5 ‚Üí 12) / `DIGEST_LESSONS_TOPN` (8) + the existing fence budget bound the cost. Fixed a latent ordering bug exposed by the new tests: same-day entries now tie-break by append order so "newest first" is honest. *Honest scope (corrected from the first cut):* what's resident is the **existence map** (titles + counts + access path), not bodies ‚Äî and retrieval is lexical, so a synonym-only match can still be missed. New optional **`- aliases: <synonyms / old names>`** on a decision/lesson (CLI `--aliases`, MCP `aliases`) adds search terms so lexical search finds an entry by words not in its title/body ‚Äî a zero-dep mitigation for the synonym gap (the whole entry is the search text, so no retrieval change needed).
2. **Relevance-decay fields (`updated:` + `code:`).** `- updated:` (last-confirmed date) is tracked separately from `- date:` (first recorded); recency ranking and the staleness flag key off it, so a re-confirmed old decision ranks fresh instead of decaying. New `reconfirm` command/MCP tool re-stamps `updated:` (and can re-raise confidence) **without rewriting** the entry. Optional `- code: <path ‚Üí symbol>` back-links the file an entry governs; `doctor` gains a `[code-links]` check that flags dangling paths (URLs and bare symbols skipped).
3. **Lesson lifecycle (mark-over-delete).** New `resolve-lesson` command/MCP tool retires a lesson as `resolved` (root cause fixed) or `obsolete` (premise gone): it leaves default search but stays reachable via `--history`, so a hard-won lesson is never lost. `isRetiredStatus()` consolidates the scattered `superseded`/`deprecated` checks and now covers `resolved`/`obsolete`; `statusWeight` adds `resolved` 0.1 / `obsolete` 0.05.
4. **Review hardening.** `upsertEntry`'s supersede scan skips *all* retired statuses (a `resolved` lesson can no longer be resurrected as a supersede target); `updateEntryMeta` validates `confidence`/`status` and refuses an ambiguous title match (lists candidates) instead of silently re-stamping the first; the grounding count uses `isRetiredStatus` consistently.
5. **Writing discipline (recoverability test).** SKILL.md + templates add the litmus test: *could a competent engineer reconstruct this from the code as it stands?* If yes, don't record it ‚Äî the ledger is for what code can't show.
6. **Docs.** README (EN + zh-TW): documented the above; corrected the long-standing "‚â§200-token digest" claim (it conflated `MAX_TRANSCRIPT_LINE=200` *chars* with a token budget ‚Äî the real bound is `DIGEST_CHAR_BUDGET=1200` chars).

## v2.8.0 (2026-06-23)

**Reliability release ‚Äî make the headline promise (compaction survival) degrade *loudly* instead of silently, and stop the write path from corrupting the ledger.** Motivated by auditing a real non-author project whose ledger had drifted; the two integrity fixes were each hardened by multi-round adversarial review (Claude + Codex).

1. **Fail-loud `TASK.md` parsing (#28).** `parseTask` only matched `NOW: ` (colon + space), so a drifted `NOW„Äêtag„Äë‚Ä¶` line silently parsed to nothing and the digest shipped `NOW: (not set)` ‚Äî the compaction-survival promise failing without a peep. Now: a tolerant matcher accepts `NOW:` / full-width `NOWÔºö` / bare `NOW ` / bracketed `NOW„Äê‚Ä¶„Äë` (lookahead so `NOWHERE` never matches); `NOW`/`NEXT` are first-wins (newest, given prepend-drift). The digest and `doctor` **warn ‚Äî before the fenced TASK body, so a re-anchoring agent reads the distrust signal first** ‚Äî when GOAL is set but `NOW` can't be parsed, when there are multiple `NOW`/`NEXT` lines (appending instead of rewriting), or when `NOW` reads like completed work (‚úì / Â∑≤Âêà‰Ωµ / Â∑≤ÂÆåÊàê). `touchTaskUpdated` now canonicalizes any parseable `UPDATED` line.
2. **ADR write integrity (#27).** The decision write path ran authored bodies through `sanitize()`, whose 200-char transcript cap (meant for untrusted transcript lines) silently truncated any Context/Decision/Consequences field over 200 chars and round-tripped a ` [‚Ä¶]` marker into the canonical ledger. `sanitize()` now takes `{authored:true}`: keeps secret/PEM redaction, replaces the 200-char marked cap with a silent 4000-char anti-bloat cap (no marker), and warns to stderr on truncation. Covers `decision` / `--body` / `write_lesson` (the shared `upsertEntry` choke point).
3. **Bounded digest + auto-consolidation + search-first protocol (#26).** `buildDigest` enforces `DIGEST_CHAR_BUDGET` on the fenced content (the per-turn digest can no longer grow with the ledger); `consolidate` gains a non-blocking `--auto-safe` path that `PreCompact` runs after snapshotting (lean ledger without masking staleness); AGENTS protocol rule mandates `coderecall search` over full-file reads of `DECISIONS.md`/`LESSONS.md`; an **auto-memory boundary rule** in the init template stops platform auto-memory (Claude/Gemini/Cursor/Cline) from duplicating the ledger (the `.ai/memory/` ledger is SSOT).
4. **Write-back reliability primitives (#19, #20).** Evidence-based staleness ‚Äî the digest's "stale" signal is now "source moved past the ledger" (a commit newer than `UPDATED`, or dirty source), not just the clock. A pre-commit advisory nudge when staged source outruns the ledger. New `check` command: a read-only gap report (how long since `UPDATED`, source commits since, uncommitted files, verdict; `--strict` exits non-zero).
5. **Write-back trustworthiness benchmark (#21).** `bench/bench.js` gains a deterministic section that drives the real CLI over a synthetic git history and reports gap-**detection** coverage (100% surfaced / 0 false-alarm on the synthetic session). Honestly scoped in-tool: this measures the detection mechanism, not whether a developer acts on the nudge.
6. **Digest token trim (#18).** SessionStart/resume digest ~13% smaller (359 ‚Üí 313 tokens) via a single untrusted-data fence + a tiered protocol line (full text only after compaction); zero content-signal loss.
7. **Docs.** Pain-first README hero ("your coding agent forgets *why*" + compaction-survival), a reproducible `docs/launch/demo.js`, the bilingual EN/zh-TW switch, the "what it fills vs. what it doesn't replace" comparison table, and the new fail-loud feature bullet (#22‚Äì#25, #29).

## v2.6 (2026-06-10)

**Surfacing + anti-re-litigation** ‚Äî keep effective decisions *influential*, not just stored.

1. **Decision surfacing in the digest.** SessionStart / post-compaction digest now injects the **top-5 current decisions** (titles only, newest-first; superseded/deprecated/expired **and `proposed`** excluded ‚Äî only accepted current truth; fenced+sanitized). Previously DECISIONS only contributed a count and never reached the agent's context; now the agent sees current decisions every session ‚Äî curated attention (top-N current), not the whole log, so no context pollution. None on a fresh ledger, so the ‚â§1200-char fresh-digest budget is unaffected.
2. **Re-litigation warning.** Recording a decision (`decision` CLI / MCP `write_decision`) that clearly overlaps an accepted decision but sits **below** the auto-supersede bar offers three resolutions: `--supersedes "X"` (replace), `--confirm-new` (it's distinct ‚Äî suppresses the note), or revise the title. Catches re-deciding that the >0.8-Jaccard auto-match misses. SPEC now states the boundary verbatim: *Code Recall does not enforce decisions against code; it keeps accepted decisions visible and warns when new decisions reopen settled ones without superseding.*

Honest boundary (stays zero-dep, no LLM/vector): detecting that *code* contradicts a decision needs semantics and is out of scope. Code Recall instead puts current decisions in front of the agent and warns at authoring time ‚Äî maximizing "seen, not re-decided" without semantic contradiction detection. selftest 23/23. 2.6.0.

## v2.5 (2026-06-10)

**Influence governance** ‚Äî the deepest gap for any long-term memory tool isn't *forgetting*, it's *wrongly remembering*: superseded/stale decisions keep getting recalled and pollute context ("influence rot"). Code Recall already had the lifecycle *model* (status) but ignored it at retrieval; this release makes it decisive. Treats decisions like **Git (what's HEAD)**, not a vector store (what's most similar).

1. **Lifecycle-aware retrieval (P1).** `search` and MCP `search_memory` now default to **current truth** ‚Äî superseded/deprecated/expired entries and the `archive/` are excluded; `--history` (or `history:true`) includes them, labeled `[superseded]`/`[deprecated]`/`[archived]`. New `decisions [--all]` command + MCP `list_decisions` tool = the HEAD view of current decisions.
2. **Explicit supersede (P2).** `decision "<new>" --supersedes "<old title/substring>"` (and MCP `write_decision { supersedes }`) retires a prior decision by substring match ‚Äî independent of title similarity, fixing the brittle >0.8-Jaccard auto-match for reworded titles.
3. **Weighted ranking (P3).** Retrieval score = BM25 √ó statusWeight (accepted 1.0 / proposed 0.5 / deprecated 0.2 / superseded 0.05) √ó confidence √ó recency. On an equal term match, current/high-confidence/recent decisions outrank stale ones.

Priority order followed the analysis: **Lifecycle ‚Üí Weighting ‚Üí Pruning** (pruning via `consolidate`/`expires` already shipped). selftest now 20/20 (added explicit-supersede + current-vs-history retrieval checks). 2.5.0.

## v2.4 (2026-06-10)

**Reliable capture** ‚Äî the answer to the decision-log's #1 risk (honor-system write-back). Two non-coercive levers, both consistent with the earlier decision *not* to use Stop-hook prompts:

1. **`coderecall decision "<title>"` CLI** ‚Äî record an ADR-grade decision from the terminal in one line (`--context`/`--decision`/`--consequences`/`--status`/`--confidence`, or `--body`). Mirrors the MCP `write_decision` tool and is supersede-aware. Lowers the friction of capturing a decision so it actually happens.
2. **Pre-commit capture nudge** ‚Äî `precommit` prints one advisory line when a commit stages real source files but records no decision ("N source file(s) staged, no decision recorded ‚Äî `coderecall decision ‚Ä¶`"). At the deliberate commit checkpoint, never blocks, never via a Stop hook. Conservative heuristic (skips docs/config/ledger-only commits).

`score` was deliberately left unchanged ‚Äî it measures working-state health and shouldn't penalize an early task for having no decisions yet. selftest now 16/16.

## v2.2 (2026-06-10)

1. **`coderecall score [--json]` ‚Äî working-state health.** Rates whether the ledger can actually drive an agent's next move, via a transparent weighted heuristic: GOAL specificity, NEXT concreteness (vague values like "continue"/"TBD" score low even when the field is "filled"), NOW set, every `[!]` blocker carrying a reason, freshness, and grounding in recorded decisions/lessons. Each dimension reports its score + why + a "fix first" list. Deliberately not an ML/precision number ‚Äî the goal is to catch *false completeness* (a ledger that looks done but can't tell an agent what to do next). `status` now prints the one-line overall. selftest gained 3 checks (concrete NEXT = 100, vague NEXT < 60, fresh template scores low) so the heuristic stays honest.
